最近想搞个番茄钟小工具放浏览器里挂着，工作的时候用。功能不用太复杂，能计时、能记历史就行。技术栈用 React + Vite，单页应用，浏览器一关数据丢了不行要能存住。

业务方的需求比较模糊。先说时长——我问了几个同事有人说"标准番茄钟就是 25 分钟啊"，但也有人说"我不喜欢标准，我自己专注时间短用 15 分钟的"。再问休息，标准是短休息 5 分钟、长休息 15-30 分钟，但 4 个番茄之后休息多久没定论；有人说"反正我累了就休息多久都行"，但这又没法程序化。最后方案是：让用户自己配置所有时长，工作时长、短休息时长、长休息时长、长休息触发频率（每完成几个番茄进入长休息）这些都可以在设置里改，初始默认值给个常见组合（25-5-15-4）。让用户能个性化就完了。

还有个隐含的模糊——统计这块。同事说"我就想知道今天做了几个番茄，简单点"，但他又要"按项目分类统计我每个项目花了多少时间"，这俩其实是矛盾的：数据简单就意味着不分类，分类就意味着记录项目字段。系统设计成"可选"模式——用户可以选不分类，统计只有总时间；或者选开分类，每个番茄记录是哪个项目，统计页面才出项目维度的数据。

数据存储用 localStorage，所有数据在浏览器里。结构大致：sessions 数组每条记录 {id, type: 'work'|'short_break'|'long_break', project?: string, started_at, ended_at, completed: boolean}。completed 字段是 timer 自然结束还是中途取消——自然完成的算一个完整番茄。

计时器功能：右上角或者页面顶部一个大大的倒计时显示，中间圆环或者进度条表示剩余时间。控制按钮：开始、暂停、继续、结束当前。结束当前会取消这一轮不算番茄。番茄结束后自动进入对应的休息时段（短/长根据累计），休息结束后可以选择"开始下一个番茄"或者"再休息一会儿"。

页面切换：番茄运行中、休息中、空闲 三种状态对应不同 UI。番茄运行中显示工作倒计时和大字"专注中"，休息中显示休息倒计时和"休息一下"，空闲显示"开始工作"。

通知：番茄结束和休息结束的时候用浏览器 Notification API 弹通知，同时播放一个轻提示音（用 Web Audio 合成一个简单的 ding 音）。这样用户切到别的标签页也能看到。Notification 在用户没授权时优雅降级为页面内提示。

历史记录页面：列出所有 session，按时间倒序。每条可以编辑（补填项目名）、删除、按日期/项目筛选。统计页面：今日番茄数、本周番茄数、连续天数（streak）、按项目饼图、最近 7 天柱状图、平均每日番茄数。统计图表用 recharts 简单画一下。

CSV 导出：把历史 session 导出成 CSV，字段 id, type, project, started_at, ended_at, duration_secs, completed。导出按钮在历史页和统计页各放一个。导出的文件命名带日期 tomato-history-2026-06-09.csv。

设置页面：四个时长字段、项目列表管理（增删改）、主题（深色/浅色）、是否开启声音、是否开启桌面通知、每周一开始还是周一开始（影响 streak 计算）。

边界处理：用户关闭浏览器再打开，未完成的 session 是恢复还是作废？默认行为——如果关的时候在 work 状态，重启后提示"上次有个未完成的番茄，要不要继续"，选是就接着原剩余时间，选否就标记为取消。如果关的时候在 idle 就没事。切换标签页/电脑休眠：番茄钟用了 setInterval 一秒一秒减时间，但浏览器后台可能会节流（mobile 上尤其明显），导致实际倒计时变慢。处理办法是用 Date.now() 和 started_at 算剩余时间而不是用 interval 累加——每次刷新都重新算真实剩余时间。

快捷键支持：空格开始/暂停，R 重置，数字键 1-5 切换主题色之类。键盘党友好。

组件拆分：Timer 组件、SessionList 组件、StatsPanel 组件、Settings 组件、NotificationManager 组件。状态用 useReducer 管。

视觉风格：莫兰迪色系，柔和不刺眼。倒计时数字用等宽字体。响应式布局手机也能用。

依赖：react、react-dom、recharts、date-fns。控制在 15 个以内。

最终能跑、能计时、能分类统计、能导出 CSV、关浏览器再开数据还在。
